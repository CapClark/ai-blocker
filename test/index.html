AI Blocker — test page

A controlled page for watching the extension work, with one mock AI surface and images that carry declared-AI metadata.

Setup
  1. Load the extension at chrome://extensions (Developer mode → Load unpacked).
  2. Serve this folder over HTTP so the worker can read the image bytes:
    cd test && python3 -m http.server 8000
  3. Open http://localhost:8000/ (the file:// scheme won't work — image bytes can't be fetched from it).

1 · Mock AI surface

With the extension on (Search category enabled), the AI Overview below is removed or collapsed. The normal result stays put — that's the no-false-positives check.

2 · Declared-AI images

Each image embeds metadata markers. With Images: Label they get a corner badge; with Blur the red "AI" ones blur until clicked. The last one is a clean control and should stay untouched.

Right-click any image and choose Verify image with AI Blocker to run the on-demand check — it reports the metadata verdict instantly, then the pixel model (a labeled stub until a model is wired in).

What the badge should read

Open the popup: the count should be 1 AI surface hidden plus 6 AI images flagged — three AI PNGs, both AI JPEGs (declared, red “AI”), and one context-inferred image (amber “AI?”). The CR and plain images don't count toward the tally. Switch Images to Blur, or allowlist localhost, and reload to see everything revert.
